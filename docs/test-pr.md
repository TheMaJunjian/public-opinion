## 什么是 PR（Pull Request）？

Pull Request（简称 PR）是 GitHub 等代码托管平台提供的一种协作机制，允许开发者将自己在独立分支上完成的代码变更，申请合并（merge）回主分支（如 `main`）。

### PR 的常见用途

1. **代码审查（Code Review）**：团队成员可以在 PR 页面逐行查看变更、提出意见或建议，确保代码质量。
2. **持续集成（CI）**：PR 通常会自动触发 CI 流水线（如 GitHub Actions），运行测试、代码检查等，确保新代码不破坏已有功能。
3. **合并变更（Merge Changes）**：经过审查和 CI 通过后，仓库维护者可以将 PR 合并到目标分支，完成代码落地。
4. **讨论与记录**：PR 本身也是一个讨论区，相关的决策、背景说明都会保留在 PR 中，方便日后回溯。

### 典型 PR 流程

1. 从主分支创建新分支（feature/fix branch）
2. 在新分支上开发、提交代码
3. 推送分支到远程仓库
4. 在 GitHub 上发起 Pull Request
5. 团队成员进行代码审查，提出修改意见
6. CI 自动运行测试
7. 审查通过、CI 绿灯后，合并到主分支
